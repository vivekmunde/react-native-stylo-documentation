Stylish

Stylish components are nothing but enhanced React Native components. Stylo adds two extra properties styleNames(ref: StyleName) & styleNamespace(ref: Namespace) to the React Native components, on top of core properties provided by React Native. Stylo does not make any behavioral changes to these enhanced stylish components and keeps them as pure as they are defined by React Native. Stylish component reads the styles defined for the styleNames under the styleNamespace in the theme, combines these styles into one to create a standard React Native style object & applies this style to the React Native component.

Stylo's objective is to provide an enhanced Stylish component against for React Native component. This way it plans to cover all the React Native components.

Each React Native component has its own style type. E.g. Text component needs style object of type StyleProp<TextStyle>, View component needs style object of type StyleProp<ViewStyle> etc. So each stylish component is tightly coupled with a React Native component which applies the style object specific to that component.

Stylish props

All stylish components have two extra props other than the default props provided by React Native.

styleNames (Required)

The StyleNames which define the styles.

styleNamespace (Optional)

The Namespace to be used which holds the style definitions for the StyleNames supplied to the hook.
If not provided then the styler hook uses a matching default namespace from the collection of it's Default Namespaces.
